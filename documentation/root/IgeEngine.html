Description
The base engine class definition.

Methods

$( item)

Returns an object from the engine's object register by the object's id. If the item passed is not a string id then the item is returned as is. If no item is passed the engine itself is returned.
  • {}item

$$( groupName)

Returns an array of all objects that have been assigned the passed group name.
  • {}groupName

register( obj)

Register an object with the engine object register. The register allows you to access an object by it's id with a call to ige.$(objectId).
  • {}obj
Returns {*}

setFps(Number fpsRate)

Sets the frame rate at which new engine ticks are fired. Setting this rate will override the default requestAnimFrame() method as defined in IgeBase.js and on the client-side, will stop usage of any available requestAnimationFrame() method and will use a setTimeout()-based version instead.
  • {Number}fpsRate

showStats(Number val, Number interval)

Gets / sets the stats output mode that is in use. Set to 1 to display default stats output at the lower-left of the HTML page.
  • {Number}val
  • {Number}interval The number of milliseconds between stats updates.

_createStatsDiv()

Creates the stats output div on the DOM.

_removeStatsDiv()

Removes the stats output div from the DOM.

Defines a class in the engine's class repository.

  • {Object}obj The class definition.

getClass(String id)

Retrieves a class by it's ID that was defined with a call to defineClass().
  • {String}id The ID of the class to retrieve.
Returns {Object} The class definition.

viewportDepth(Boolean val)

Gets / sets the flag that determines if viewports should be sorted by depth like regular entities, before they are processed for rendering each frame. Depth-sorting viewports increases processing requirements so if you do not need to stack viewports in a particular order, keep this flag false.
  • {Boolean}val
Returns {Boolean}

dependencyTimeout( val)

Sets the number of milliseconds before the engine gives up waiting for dependencies to be satisfied and cancels the startup procedure.
  • {}val

textureLoadEnd()

autoSize( val)

Gets / sets the _autoSize property. If set to true, the engine will listen for any change in screen size and resize the front-buffer (canvas) element to match the new screen size.
  • {}val

Returns {*}

createFrontBuffer(Boolean autoSize, Boolean dontScale)

Creates a front-buffer or "drawing surface" for the renderer.
  • {Boolean}autoSize Determines if the canvas will auto-resize when the browser window changes dimensions. If true the canvas will automatically fill the window when it is resized.
  • {Boolean}dontScale If set to true, IGE will ignore device pixel ratios when setting the width and height of the canvas and will therefore not take into account "retina", high-definition displays or those whose pixel ratio is different from 1 to 1.

canvas( elem, autoSize)

  • {}elem
  • {}autoSize

clearCanvas()

openUrl( url)

Opens a new window to the specified url. When running in a native wrapper, will load the url in place of any existing page being displayed in the native web view.
  • {}url

showWebView(String url)

Loads the specified URL as an HTML overlay on top of the front buffer in an iFrame. If running in a native wrapper, will load the url in place of any existing page being displayed in the native web view. When the overlay is in use, no mouse or touch events will be fired on the front buffer. Once you are finished with the overlay, call hideOverlay() to re-enable interaction with the front buffer.
  • {String}url

Hides the web view overlay.
Returns {*}

watchStart(* evalStringOrObject)

Adds a new watch expression to the watch list which will be displayed in the stats overlay during a call to _statsTick().
  • {*}evalStringOrObject The expression to evaluate and display the result of in the stats overlay, or an object that contains a "value" property.
Returns {Integer} The index of the new watch expression you just added to the watch array.

watchStop(Number index)

Removes a watch expression by it's array index.
  • {Number}index The index of the watch expression to remove from the watch array.

traceSet(Object obj, String propName, Number sampleCount)

Sets a trace up on the setter of the passed object's specified property. When the property is set by any code the debugger line is activated and code execution will be paused allowing you to step through code or examine the call stack to see where the property set originated.
  • {Object}obj The object whose property you want to trace.
  • {String}propName The name of the property you want to put the trace on.
  • {Number}sampleCount The number of times you want the trace to break with the debugger line before automatically switching off the trace.

traceSetOff(Object object, String propName)

Turns off a trace that was created by calling traceSet.
  • {Object}object The object whose property you want to disable a trace against.
  • {String}propName The name of the property you want to disable the trace for.
